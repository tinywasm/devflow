# PLAN: Badge Markdown Simplification

Refactor badge README injection to eliminate section markers and minimize token consumption.

## Goal

Replace the current verbose badge markdown (with `<!-- START_SECTION -->` / `<!-- END_SECTION -->` markers and `<a><img></a>` wrapper) with a single minimal line:

```html
<img src="docs/img/badges.svg">
```

All metadata ("Generated by devflow") stays inside the SVG file only.

## Architecture Decision

**Approach B — Badge-specific logic in badges.go**, bypassing `UpdateSection` for README updates. The generic `markdown_updater.go` remains unchanged for other sections.

Rationale: Badges are the only section that needs markerless detection. Adding pattern-matching to the generic updater would complicate it for a single use case. SRP: badges.go owns its own README update strategy.

## Stages

### Stage 1: Simplify BadgeMarkdown output

**File: badges.go**

1. Change `BadgeMarkdown()` (line ~207) to return only:
   ```go
   return fmt.Sprintf(`<img src="%s">`, h.outputFile)
   ```
   Remove `<a>` wrapper, `alt`, `title`, and `badgesInfo` reference.

2. Remove `badgesInfo` field from `Badges` struct and `NewBadges()` initialization (it was only used in BadgeMarkdown). Keep `svgInfo` for the SVG comment.

### Stage 2: Badge-specific README updater

**File: badges.go**

Add a new method `UpdateReadme()` that replaces both `BuildBadges` return-args-for-UpdateSection pattern and the `updateBadges` call to `MarkDown.UpdateSection`.

Logic:
1. Read the README file
2. **Detect old format**: Find lines matching `<!-- START_SECTION:BADGES_SECTION -->` and `<!-- END_SECTION:BADGES_SECTION -->` — remove the markers AND everything between them
3. **Detect current format**: Find line matching pattern `<img src="` + `badges.svg">` (contains "badges.svg" inside an img tag)
4. **Replace or insert**: If old markers found, replace block with the single `<img>` line at the same position. If existing img line found, replace it. If neither found, insert after line 1 (title line).
5. Write the file only if content changed.

Update `BuildBadges()` to call `UpdateReadme()` directly instead of returning section args. Change return type from `([]string, error)` to `error` since the args are no longer needed externally.

Update `updateBadges()` to use the new `UpdateReadme()` instead of calling `MarkDown.UpdateSection`.

### Stage 3: SVG comment improvement

**File: badge_svg.go**

Change the SVG comment (line ~129) from:
```

```
To include the target module context:
```
<!-- Generated by github.com/tinywasm/devflow -->
```

Simplify `svgInfo` to just use `DevFlowRepository`. The SVG is self-documenting — anyone viewing the source sees where the tool lives.

### Stage 4: Update tests

**File: test/markdown_updater_test.go** — No changes needed (generic updater unchanged).
